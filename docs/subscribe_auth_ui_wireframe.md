## 2. 无密码登录/身份弹窗交互 (Auth Modal)
当用户未登录，但试图在 Open Lab 里**发表评论**或**提出具体 Feature** 时触发。

```text
[ 居中悬浮弹窗 (Modal) ]
--------------------------------------------------
[ 锁 Icon ]
想要参与共创？留个邮箱就行。

为了保证社区质量，我们需要验证一下这是你本人的邮箱。
不需要设置烦人的密码。

[ 你的称呼 (选填，默认是“某位极客”) ]
[ 你的邮箱 (必填)                  ]

[ 获取 6 位验证码 ]

(若点击获取验证码，下方滑出输入框)
[ 输入邮箱收到的 6 位验证码...      ]

[ 确认身份并发布 ]

(小字：验证后你将在本设备保持长久登录态)
--------------------------------------------------
```
* **交互细节**：必须通过 OTP（一次性密码）验证，才能在后台创建 User 记录并发出评论。这彻底解决了“乱填别人邮箱导致骚扰或冒名顶替”的问题，同时依然保持了无密码的顺滑体验。
