# AI 创造与实验专题 (AI Experiments) UI/UX 交互线稿与视觉规划

## 1. 整体视觉风格 (Visual Tone)
*   **风格基调**：与 Open Lab 保持一致的 **Geek 极简风**，但比 Lab 的“生猛”稍微收敛一点，更偏向“杂志/画册”的阅读体验。
*   **排版逻辑**：左（导航）- 中（内容瀑布）- 右（工具箱/导流）的 Multi-shell 结构，滚动时左右两边吸顶不动。

---

## 2. 页面结构线稿 (Wireframe)

### 2.1 顶部横幅 (Hero Section)

```text
[ 巨型粗体标题 ] AI 使用经验
[ 副标题 ] 这里没有水文，只有从“公开实验”里蹚出来的真金白银。涵盖产品复盘、Vibe Coding 技巧与前沿模型测评。
```

### 2.2 核心双栏结构 (Two-Column Layout)
抛弃繁杂的三栏，采用更聚焦的 **“左侧导航字典 + 右侧干货瀑布流”** 结构。
左侧导航为动态读取，不写死分类，随着内容的丰富可随时在后台新增大类。

```text
[ 左侧栏 (Dynamic Nav) - 吸顶 ]     [ 右侧主内容区 (Main Content Stream) ]
                                |
📍 分类导航 (后台动态配置)         |   ==== 🛠️ 实战复盘 (Build in Public) ====
                                |   [卡片] 视频：我是如何用 Cursor 3天做完 xxx 的？
- 🛠️ 实战复盘                     |   [卡片] 文章：Open Lab 需求孵化器 MVP 落地全复盘
- ⚡ 工作流与技巧                  |   [卡片] 文章：独立开发者如何做支付接入？
- 🧠 模型测评与前沿                |
- [未来可能新增的分类...]           |   ==== ⚡ 工作流与技巧 (Workflows) ====
                                |   [卡片] 仓库：一套拿来即用的 Next.js 极简模板
👇 没找到想要的？                 |   [卡片] 视频：Vibe Coding 进阶：如何写出让 AI 听话的 Prompt
[ 去 Open Lab 提需求 🚀 ]        |   [卡片] 文章：如何用 AI 自动化我的 RSS 信息流
                                |
                                |   ==== 🧠 模型测评与前沿 (Reviews & Frontier) ====
                                |   [卡片] 文章：Gemini 1.5 Pro 深度测评：它真的能取代 Claude 吗？
                                |   [卡片] 视频：2026 个人开发者还值得做套壳站吗？
```

### 2.3 内容卡片细节 (Card Detail)
为了提升专业感，卡片需要展示“核心干货”，让用户不用点进去就知道值不值得看。

```text
------------------------------------------------------------
[ 🏷️ Video | 📅 2026-02-28 ] 
大字标题：Vibe Coding 进阶：如何写出让 AI 听话的 Prompt

[摘要/Takeaways 模块 - 带背景色]
👉 核心结论：
1. 不要教 AI 怎么写代码，告诉它你的系统架构。
2. 给出“坏例子”比给出“好例子”更有约束力。
3. 附带了一套我的 .cursorrules 模板。

[ 播放视频 ↗ ]   [ 阅读文字版 ↗ ]   |   [ ☕️ 觉得有用？请我喝咖啡 ]
------------------------------------------------------------
```
